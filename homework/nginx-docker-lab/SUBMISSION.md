# Submission Notes

## What to include

- Screenshots from `screenshots/` (files `01` through `08`)
- GitHub repository link in the email body
- Clear subject line using the course submission format

## Screenshot checklist

| File | Verify |
|------|--------|
| `01-ec2-instance-and-security-group.png` | EC2 instance is running; security group configured |
| `02-ssh-connection-to-ec2.png` | SSH connection to the instance works |
| `03-docker-installed-and-tested.png` | Docker is installed successfully |
| `04-docker-run-nginx-container.png` | Nginx container started with `docker run` |
| `05-nginx-container-docker-ps.png` | `docker ps` shows the Nginx container |
| `06-nginx-welcome-page-browser.png` | Nginx welcome page opens in the browser |
| `07-docker-run-and-cleanup.jpg` | Container stopped/removed after validation |
| `08-ec2-instance-terminating.png` | EC2 instance terminated after cleanup |

## Best practice

- Keep screenshot names clear and consistent (numbered `01-` … `08-` prefix).
- Do not commit secrets or private keys (`.pem`, access keys).
- Keep the email short, professional, and friendly.
